Hostify

A place to put your HTML on the internet. Nothing more.

Why

AI models got good at generating HTML. You describe a page, you get a working site in seconds. But then you have an HTML file on your computer with no simple way to show it to anyone.

For a non-technical person who just asked ChatGPT to make them a portfolio, the path from "file on desktop" to "link I can share" is surprisingly hard. You need a hosting account, or git knowledge, or to figure out Netlify/Vercel/GitHub Pages.

Hostify removes all of that. Pick a name, upload your file, get a URL. The simplest path from HTML file to live webpage.

How

Submissions are stored in a public GitHub repo. When you upload, a PR is created. An automated CI pipeline checks for security issues (redirects, iframes, miners, phishing forms).

If checks pass, the file merges and is instantly live at a permanent URL. Your page renders inside a sandboxed iframe, so it runs JavaScript freely without affecting anything else.

Open source. Transparent rules. No black box. One file per page, self-contained HTML only, max 500KB.

Not a website builder

You bring your own HTML. AI-generated, hand-written, exported from anywhere. Hostify just hosts it.

Not a full platform

One file per page. No databases, no server code, no multi-file projects. Just a single HTML file.

Not anonymous

You pick a username. Pages live under your name. There's a review step before anything goes public.

Who it's for

  • → The person who asked AI to make a landing page and doesn't know what to do next
  • → The developer sharing a quick demo without setting up infrastructure
  • → The student learning HTML who wants their first page on the real internet
  • → Anyone with a single HTML file who wants the world to see it

Rules

Allowed

  • ✓ Any CDN (Tailwind, GSAP, etc.)
  • ✓ External images/fonts
  • ✓ Canvas, WebGL, SVG
  • ✓ JavaScript
  • ✓ Up to 500KB

Blocked

  • ✗ Iframes
  • ✗ Redirects
  • ✗ External form actions
  • ✗ Crypto miners
  • ✗ Meta refresh
